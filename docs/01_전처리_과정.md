## 처리 단계

```
PDF 파일 → PDF 파싱 → CSV 매칭 → Rule 매핑 → 수학 검증 → parquet 저장
```

### 1. PDF 파싱
- pdfplumber로 테이블 추출
- 버전 식별 (v2.0~v4.1)
- 카테고리별 점수 + 인증 등급 추출

### 2. CSV 매칭
- PDF의 project_id 또는 건물명을 CSV와 매칭
- 건물 메타 정보(연면적, 용도) 보강

### 3. Rule 매핑
- `data/rubrics/mapping_rules.yaml` 의 **107개 규칙** 적용
- 버전별 카테고리 → v5 카테고리로 변환
- SS→LT 분리 (v2.x/v2009), IN/RP 폐지, IEQ→EQ

### 4. 수학 검증
- 드리프트(원본 달성률 vs v5 달성률) ≤ 20%
- 카테고리 점수 ≤ v5 최대값

### 5. parquet 저장
- `data/project_features.parquet` (460행)
- `data/standardized_credits.parquet` (9,747행)
