# 03. 분석 방법론 (XGBoost + SHAP)

## 연구 질문

**한국 LEED 건물에서 어떤 카테고리가 등급을 결정하는가?**

## 모델

- **모델**: XGBoost Classifier
- **하이퍼파라미터**: n_estimators=200, max_depth=6, learning_rate=0.05
- **Feature**: 9개
  - 카테고리 달성률 7개: `ratio_LT, ratio_SS, ratio_WE, ratio_EA, ratio_MR, ratio_EQ, ratio_IP`
  - `log_area` (연면적 log)
  - `version_ord` (버전 순서 인코딩)
- **Target**: `certification_level` (Certified/Silver/Gold/Platinum)

## 검증

- **5-Fold Stratified CV** (등급 분포 보존)
- NaN은 `fillna(0)` (v2009/v2.2의 ratio_SS 141건)

## 성능

| 지표 | 값 |
|------|-----|
| **CV Accuracy** | **0.8174 ± 0.0502** |
| **CV Weighted F1** | **0.8157 ± 0.0504** |
| Train Accuracy | 1.0 (과적합 — 논문 미사용) |

## SHAP Top 5

| 순위 | Feature | 평균 \|SHAP\| |
|------|---------|-------------|
| 1 | **ratio_EA (에너지)** | **0.8840** |
| 2 | ratio_EQ (실내환경) | 0.6533 |
| 3 | ratio_WE (물) | 0.5895 |
| 4 | ratio_LT (입지/교통) | 0.4377 |
| 5 | version_ord | 0.2880 |

**해석**: 한국 LEED 등급은 **에너지(EA)가 가장 크게 결정**. 설계 단계 EA 확보가 등급 업그레이드 효과적.

## Robustness

| Subset | N | CV Acc | Top SHAP |
|--------|---|--------|----------|
| 전체 | 460 | 0.8152 | ratio_EA |
| credit_hit > 0.7 | 324 | 0.8333 | ratio_EA |
| 구버전 (v2.x, v2009) | 136 | 0.7865 | ratio_EA |

→ **어느 subset이든 EA가 Top**. 결과 robust ✅

## 심사 예상 Q&A

### Q1. 왜 LangGraph? 함수로 해도 되잖아요

각 단계가 **독립 노드로 분리**돼 디버깅 쉬움. 조건부 엣지로 분기 정책 한곳에 집중. State로 중간 결과 전부 추적. 체크포인트로 8시간 실행 중 끊겨도 resume 가능.

### Q2. 왜 Rule 먼저고 LLM 나중?

1. **재현성**: Rule은 동일 입력 = 동일 결과. LLM은 매번 다름.
2. **비용**: LLM 주계산자면 수백 달러. Rule은 API 0회.
3. **책임소재**: 계산=Rule, 검증=LLM 분리 → 수치 출처 명확.

### Q3. LLM이 Rule을 90% 거부했는데 Rule이 틀린 거 아니냐?

- threshold 0.8이 보수적이라 거부율 높음 (75건 실험 기준)
- LLM 승인 7건은 **구체적 도메인 근거** 제시 ("LT/SS 분리 올바름, IN/RP 폐지 정확히 반영")
- Option A 채택으로 **점수는 항상 Rule**, LLM은 uncertainty signal만 제공

### Q4. 460개로 일반화 되나?

USGBC Public LEED Project Directory 기준 **한국 전수**. 표본 추출 아니라 한국 시장 대표. 해외 확장 시 LLM 프롬프트 국가 맥락만 교체.

### Q5. Train 정확도 1.0은 과적합 아닌가?

맞음. Train 1.0 / CV 0.8174로 gap 18%. 그래서 논문엔 CV 값만 사용. Holdout test set 미도입 — **후속 과제**로 명시.

### Q6. v2009 114건은 왜 크레딧 분석에서 빠졌나?

v2009 PDF가 **카테고리 합계만 제공**, 크레딧 상세 없음. `category_proportional`로 카테고리 레벨 분석엔 포함, 크레딧 레벨 SHAP에서만 제외.

### Q7. SS→LT 분리 비율의 출처는?

v2.2/v2009의 SS에 교통 크레딧 약 7점 포함 (LEED 공식 루브릭). 하드코딩된 값이라 **Peer review 필요** — 한계로 명시.

### Q8. LLM 리뷰 75건이 전체 대표?

16.3% 커버리지. v4가 53건으로 편중됨. 논문엔 **"표본 리뷰"**로 포지셔닝. 후속: threshold 완화 + 전수 리뷰.

### Q9. 매핑 규칙 107개는 누가 검증?

LEED 공식 루브릭 기반 연구자 수동 작성. **Peer review 미실시**. credit_rule_hit_rate로 커버리지 정량화 (v4 88%, v2009 64%).

### Q10. CV fold 분산 큰 이유?

fold별 76~89%. 원인: 구버전 표본 수 차이, 건물 유형 다양성, 460건이 feature space 대비 작음. 후속: 층화 기준에 버전 추가.

### Q11. 이 연구의 기여?

1. **버전 전수** (v2.0~v4.1)
2. **LangGraph 이중 검증** 재현 가능한 파이프라인
3. **한국 전수 460건** 공개 데이터셋
4. **SHAP 크레딧 단위** 상세 분석

### Q12. 후속 연구 방향?

1. Holdout test set + 외부 검증
2. LLM 전수 리뷰 (threshold 완화)
3. 추가 feature (용도, 위치, 준공 연도)
4. 해외 데이터셋 확장
5. 구버전 매핑 정교화
